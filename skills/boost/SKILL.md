# Boost — 세션 부트스트랩

새 세션에서 이전 작업 맥락을 복원한다. 아래 순서대로 수행:

1. 프로젝트 루트의 `GEMINI.md`를 읽는다 (있으면).
2. `work.md`를 읽는다 — 특히 최근 작업 이력, 할 일, `## AntiPatterns`, 의사결정 로그.
3. `git log --oneline -10`과 `git status --short`로 최근 커밋과 작업 트리 상태를 확인한다.
4. 위 정보를 종합해 다음 형식으로 보고한다:

```
## 현황
- 마지막 작업: (work.md + 최근 커밋 기준 한두 문장)
- 작업 트리: (커밋 안 된 변경 유무와 핵심 파일)
- 다음 할 일: (work.md의 할 일 목록 상위 항목)
- 주의사항: (AntiPatterns나 미해결 이슈 중 현재 작업과 관련된 것만)
```

5. 보고 후 사용자 지시를 기다린다. 묻지 않고 작업을 시작하지 않는다.

## 규칙
- work.md가 없으면 git 히스토리만으로 요약하고, work.md 생성을 제안한다.
- work.md 내용이 코드 현재 상태와 다르면(예: 완료됐다고 적힌 기능이 실제로 없음) 그 불일치를 명시한다.

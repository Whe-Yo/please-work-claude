# Feedback — 사용 중 피드백·이슈 보고

이 하네스를 다른 프로젝트에 적용해 쓰다가 에이전트나 사용자가 버그·개선점·마찰을 발견했을 때, 저장소로 보고한다. 하네스 저장소는 성역(직접 작업 금지)이므로, 피드백은 **GitHub 이슈**로 보낸다.

저장소: `https://github.com/Whe-Yo/please-work-harness/issues`

## 절차

1. **내용 정리**: 아래를 모은다. 추측 말고 실제 일어난 것만.
   - 무엇을 하려 했나 / 무슨 일이 일어났나 / 기대한 것
   - 관련 스킬·규칙 (예: `antithesis`, AGENTS.md 9절)
   - 환경 (에이전트 종류, OS, 관련 도구 유무)
   - 재현 절차 (가능하면)
   - 유형: `bug` / `enhancement` / `friction`(쓰기 불편)

2. **환경별 제출 분기**:
   - **GitHub API/CLI + 인증 토큰이 가용**: 위 저장소에 이슈를 직접 생성한다.
     - 예(API): `curl -s -X POST -H "Authorization: token $TOKEN" https://api.github.com/repos/Whe-Yo/please-work-harness/issues -d '{"title":"...","body":"...","labels":["bug"]}'`
   - **불가(토큰 없음·오프라인 등)**: 아래 형식으로 **붙여넣기용 이슈 본문**을 출력해 사용자가 직접 등록하게 한다.

3. **이슈 본문 형식**:

   ```
   ## 유형
   bug | enhancement | friction

   ## 요약
   (한 줄)

   ## 무슨 일이
   - 하려던 것:
   - 일어난 것:
   - 기대한 것:

   ## 환경
   - 에이전트 / OS / 도구:

   ## 재현
   1.
   2.

   ## 관련 스킬·규칙
   (예: antithesis, AGENTS.md 9절)
   ```

## 규칙
- 이슈에 비밀(.env 값, 토큰, 자격증명)을 절대 포함하지 않는다. 토큰은 환경변수로만 참조.
- 하네스 저장소 파일을 직접 수정하지 않는다 (성역). 보고는 이슈로만.
- 단순 의견이 아니라 재현 가능한 사실 위주로. 추측은 추측이라 표시.

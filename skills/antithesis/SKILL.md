# Antithesis — 독립 컨텍스트 안티테제 검토

이전 대화 흐름에 오염되지 않은 새 AI 인스턴스가 RPW 문서만을 기반으로 검토한다.
목적: 현재 대화에서 쌓인 컨텍스트 오버피팅과 확증 편향으로 놓친 논리적 오류를 탐지.
적용 범위: 코드·설정·문서 수정, 조사·분석 결론, 설계 결정 — 단순 답변을 제외한 모든 주요 작업.

## 절차

1. **컨텍스트 수집**: RPW 문서(rule_plan_work.md 또는 work.md) 전체 + 검토 대상(diff, 코드, 설계안, 조사 결론).

2. **격리된 인스턴스 생성** (환경별):
   - **서브에이전트 툴이 있는 경우** (예: Claude Code의 Agent 툴): 현재 대화 컨텍스트 없이 새 인스턴스를 소환. 아래 검토 프롬프트만 전달.
   - **위 수단이 없는 경우**: 사용자에게 새 채팅 창을 열고 아래 프롬프트를 붙여넣도록 안내.

3. **검토 프롬프트** (이 내용만 새 인스턴스에 전달):

   ```
   [프로젝트 컨텍스트]
   {RPW 문서 전체 내용}

   [검토 대상]
   {diff 또는 코드 또는 설계안}

   [지시]
   위 내용을 검토하라.
   - 접근 방향 자체가 틀렸을 가능성을 열어두고 판단하라.
   - 논리적 비약, 누락된 케이스, 과도한 가정, 결론의 근거 부족을 찾아라.
   - 칭찬 없이 문제점과 근거만 보고하라.
   - 발견 없으면 "발견 없음"으로만 끝내라.
   ```

4. **결과 통합**: 발견된 문제를 현재 세션으로 가져와 적용 여부를 판단하고, 채택·기각 결정을 RPW 문서에 기록한다.

## 검토 횟수와 정지 조건
변경 규모로 초기 가늠(소규모 1 / 중규모 2 / 대규모 3회)하되, **횟수는 안전망일 뿐 정지의 1차 기준이 아니다.** 진짜 변수는 라운드 수가 아니라 **수렴이냐 발산이냐**다.

- **최소 1회**는 반드시 실행한다 (0회 금지).
- **1차 정지 = 수렴**: 직전 라운드 대비 **미해결 비사소 지적 수가 줄지 않으면 중단**한다. "발견 없음"이면 당연히 종료. 줄지 않는다는 건 수렴 실패(발산·오실레이션) 신호다 — 라운드를 더 돌려도 안전해지지 않는다.
- **횟수 상한 = 안전망(2차)**: 기본 3회, 절대 상한 5회. 사용자가 값을 지정하면 우선.
- **발산 탐지 시**: 마지막 라운드 산출물이 새 문제를 안고 멈췄을 수 있다. **가장 안전했던 라운드의 산출물(롤백 후보)을 보존**하고, 그것 + 미해결 지적을 사용자에게 제시한다. 최신본을 무조건 채택하지 않는다.
- **"사소·주관적" 판정은 독립 검토 인스턴스가 한다.** 작업/사용자 인스턴스가 자기 결과물에 대한 지적을 스스로 "사소"라며 기각하지 않는다(확증편향 = antithesis가 막으려는 것). 조작적 정의: **동작·정합성에 영향 없으면 사소.**
- **사용자 중재 = 기본 출구가 아니라 예외 경로.** 수렴이 정상 종료다. 발산으로 상한에 도달했을 때만 사용자에게 핵심 지적을 제시해 취사선택을 받는다.

## 규칙
- 이전 대화 내용을 검토 프롬프트에 포함하지 않는다. RPW + 검토 대상만.
- 검토자가 제안한 수정을 무비판적으로 적용하지 않는다 — 현재 맥락에서 타당성을 판단한 뒤 적용.
- 각 라운드는 직전 지적을 반영한 **수정된 결과물**을 검토한다(토론 기록 아님 — 격리 유지).

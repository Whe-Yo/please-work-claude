# Claude Peer — Claude CLI 위임

고차원 분석이 필요할 때 Claude CLI를 헤드리스 모드로 호출한다.

## 호출 방법

```bash
claude -p "질문 내용" --output-format text
```

- 파일 컨텍스트가 필요하면 질문에 파일 경로를 명시한다 (Claude가 직접 읽음). 짧은 코드면 질문에 인라인으로 포함해도 된다.
- 긴 분석은 타임아웃을 넉넉히 (5분+).
- 읽기 전용 분석이 목적이므로 Claude에게 수정을 시키지 않는다 — "분석/제안만, 파일 수정 금지"를 프롬프트에 명시.

## 질문 작성 규칙

좋은 답을 받으려면 질문에 다음을 포함한다:
1. **목표**: 무엇을 하려다 막혔는지
2. **현상**: 에러 메시지·로그 원문 (요약하지 말고 그대로)
3. **시도한 것**: 이미 실패한 접근과 실패 이유 (중복 제안 방지)
4. **제약**: 환경, 버전, 바꿀 수 없는 부분

## 결과 처리
- Claude의 답을 그대로 적용하지 말고, 현재 코드와 대조해 검증 후 적용한다.
- 채택한 결정은 `work.md`에 WHAT/WHY/REJECTED로 기록하고 출처를 "Claude 분석"으로 남긴다.
- 토큰 제한(rate limit / usage limit) 오류가 나오면 즉시 중단하고 사용자에게 보고한다. 재시도 루프 금지.

## 사용 기준
- 사용: 아키텍처 트레이드오프, 2회 이상 실패한 디버깅, 수학·알고리즘 검증
- 미사용: 단순 문법 질문, 검색으로 해결되는 것, 파일 읽으면 알 수 있는 것

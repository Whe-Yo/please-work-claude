# Paper — 학술 논문 작업

## LaTeX 수정 시
- 수정 후 컴파일 가능성을 확인한다: 환경(`\begin`/`\end`) 짝, 중괄호 짝, 특수문자 이스케이프(`%`, `&`, `_`).
- 가능하면 `pdflatex`/`latexmk`를 비대화 모드(`-interaction=nonstopmode`)로 실행해 실제 컴파일을 검증한다.
- 그림/표 레퍼런스(`\ref`, `\cref`)가 존재하는 `\label`을 가리키는지, 인용(`\cite`)이 .bib에 있는지 확인한다.

## 글쓰기 원칙
- 주장 → 근거 → 결론 흐름을 지킨다. 근거 없는 주장 문장을 만들지 않는다.
- 용어 일관성: 같은 개념에 같은 용어. 논문 내 기존 표기(notation)를 먼저 파악하고 따른다.
- 수동 표현 남발, "very", "novel" 같은 빈 수식어 지양. 간결하고 정확한 학술 영어.
- 기존 문장을 고칠 때는 저자의 문체를 유지한다. 전면 재작성은 요청받았을 때만.

## 리뷰 시 점검 항목
1. 논리 비약: 실험 결과가 주장을 실제로 뒷받침하는가
2. 표/그림: 본문 수치와 표 수치 일치, 캡션 자기완결성
3. 관련 연구: 비교 대상 누락, 인용 형식 일관성 (\citet vs \citep)
4. 재현성: 하이퍼파라미터·세팅 기술 누락

## 보고
- 수정안은 diff 형식 또는 "원문 → 수정문" 쌍으로 제시. 이유를 한 줄씩.
- 사소한 오타는 묶어서, 논리·구조 문제는 개별로 보고.
